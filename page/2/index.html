06-es6新增set内置对象

发表于 2022-12-11 分类于 es6

Set内置对象

Set是ES6扩展内置对象

Set数据结构类似于数组，但是成员都是唯一的，没有重复值

Set函数可以接受一个数组（或者具有 iterable 接口的其他数据结构）作为参数，用来初始化。

  • 向 Set 加入值的时候，不会发生类型转换，所以5和"5"是两个不同的值。

  • Set 内部判断两个值是否不同，使用的算法叫做“Same-value-zero equality”，它类似于精确相等运算符（===），

    主要的区别是向 Set 加入值时认为NaN等于自身，而精确相等运算符认为NaN不等于自身。

01-JS严格模式

发表于 2022-12-04 分类于 javaScript

严格模式

ES5中加入了严格模式(IE10以上开始支持)

​ 严格模式(strict mode)对JS语法做出了限制性规范

​ 1.消除了JS语法的不合理不严谨之处

​ 2.消除了代码运行的不安全之处

​ 3.提高编译器效率

​ 4.禁用了在ECMAScript未来版本中可能会定义的语法，为未来新版本JS做好铺垫

严格模式对正常语义做出了更改

  1. 严格模式通过抛出错误来消除了一些原有静默错误。
  2. 严格模式修复了一些导致 JavaScript引擎难以执行优化的缺陷：有时候，相同的代码，严格模式可以比非严格模式下运行得更快。
  3. 比如在ES6中新增的一些保留字如class、export等在ES5严格模式中不能用作变量名

原型链

JavaScript 常被描述为一种基于原型的语言 (prototype-based language)——每个对象拥有一个原型，对象以其原型为模板、从原型继承方法和属性。原型对象也可能拥有原型，并从中继承方法和属性，一层一层、以此类推。这种关系常被称为原型链 (prototype chain)

03-DOM文档对象模型

DOM文档对象模型

DOM文档对象模型是用来呈现以及与任意 HTML 或 XML文档交互的API。DOM 是载入到浏览器中的文档模型，以节点树的形式来表现文档，每个节点代表文档的构成部分（例如:页面元素、字符串或注释等等）。

DOM中的节点组织为节点树

DOM的顶层对象为 document

DOM中的元素分为三种

  • 文本节点
  • 元素节点
  • 属性节点

可以获取文档对象模型中的节点进行操作（对html中的标签进行操作）

  • 创建节点
  • 删除节点
  • 添加事件绑定等
