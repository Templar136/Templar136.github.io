18-JS模块化规范

发表于 2022-12-04 分类于 javaScript

JavaScript模块化规范

逐步演化：

AMD -> CMD -> CommonJs -> UMD -> ES6

AMD

  • 异步模块定义规范（Asynchronous Module Definition），人如其名，是异步加载的
  • 是运行于浏览器之上的(异步加载可防止加载模块时间过长导致浏览器假死)
  • 此规范只有一个 API，即 define 函数：define(id?, dependencies?, factory);
  • requireJs 就是它的实现

CMD

  • 通用模块定义规范（Common Module Definiton）
  • 更简单，异步加载脚本，兼容 Nodejs
  • 定义格式: define(factory);
  • SeaJS 是它的实现

CommonJs （重要）

  • 通常用于服务端的 Nodejs 中
  • 同步加载模块
  • 每一个文件就是一个模块
  • module 代表当前模块，module.exports 是对外的接口; require 方法用于加载模块

ES6

  • ES6 终于从语言层面定义了模块化规范
  • 通用：浏览器和服务器
  • import 加载模块，export 导出模块成员

浏览器中的同源政策

同源指的是三个相同

  • 协议相同
  • 域名相同
  • 端口相同

当两个网页的URL中上述任意一个不同，则两个网页不同源

  • 不同源的网页被限制不能访问修改对方网页，即不同源的网页不能通信

JS函数的形参与传入实参的关系

  • 函数中红框所示赋值语句及其之后的操作均为对函数局部变量arr的修改
    • concat方法返回一个新的数组对象，执行此语句时及其之后的局部变量arr与全局变量num没有关系
