Element-UI

Element-UI：一套为开发者、设计师和产品经理准备的基于 vue2.0 的桌面组件(component)库

Element-Plus：~基于vue3.0的组件库

使用它可以让程序员专注于项目的业务逻辑，省下了美化样式的工作量

官网:https://element.eleme.cn/#/zh-CN

使用

基于命令行手动安装Element-ui (vue2.0)

  • 在项目根目录下安装依赖包
  • 在项目的main.js文件中导入相关资源
    • 组件相关样式需要单独导入
  • 在Element-UI官网找到写好的组件代码复制
    • 此处复制一段按钮的组件代码
  • 添加到vue单文件组件App.vue中
  • 重新执行serve命令查看效果

基于命令行手动安装Element-Plus (vue3.0)

vue3.0下不能使用Element-ui，要使用Element-Plus

  1. 安装依赖包
  • npm install element-plus - S
  1. 在main.js文件中进行配置

基于图形化界面安装Element-UI

  • 查找插件

  • 配置插件按需导入
    • 红框为按需导入
    • 默认为全部导入(导致项目体积大)

  • 测试效果

JS执行机制

1.JS是单线程

  • JavaScript语言的一大特点就是单线程，同一时间只能做一件事。

这是因为JavaScript诞生的目的所致——JavaScript是为了处理页面中用户的交互，以及操作DOM而诞生的。

我们对一个DOM元素进行操作时，不能同时进行多种操作，而只能先添加，再修改，然后删除。

  • 单线程意味着任务需要排队，前一个任务结束，后一个任务才开始。这样如果JS的执行时间过长就会导致页面渲染不连贯。导致页面渲染呈现阻塞的感觉。
