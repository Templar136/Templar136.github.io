01-浏览器地址栏输入URL回车会发生什么

发表于 2021-04-13 更新于 2022-12-11 分类于 浏览器

浏览器地址栏输入URL回车会发生什么

  1. 寻找URL中域名对应的ip地址，先寻找缓存，若缓存中没有则查询DNS服务器解析DNS得到ip地址
  2. 浏览器根据ip地址和相应的端口号，构造HTTP请求
  3. 将HTTP请求发送到服务器，将HTTP请求封装在TCP包中，建立TCP连接后依次经过 传输层 网络层 数据链路层 物理层 到达服务器端，服务器对接受的数据进行层层解封后获得HTTP请求
  4. 服务器解析HTTP请求后将响应数据（HTML文档）返回给客户端浏览器，传输过程即为第三步的逆过程
  5. 浏览器根据这个html文档来构建DOM树，在dom树的构建过程中如果遇到JS脚本和外部JS连接，则会停止构建DOM树来执行和下载相应的代码，这会造成阻塞，所以建议js文件和代码放在html代码后面
  6. 之后根据外部样式，内部样式，内联样式构建一个CSS对象模型树CSSOM树，构建完成后和DOM树合并为渲染树，这里主要做的是排除非视觉节点，比如script，meta标签和排除display为none的节点，
  7. 之后进行布局，布局主要是确定各个元素的位置和尺寸，
  8. 最后是渲染页面，因为html文件中会含有图片，视频，音频等资源，在解析DOM的过程中，遇到这些都会进行并行下载，浏览器对每个域的并行下载数量有一定的限制，一般是4-6个

03-浏览器回流(Reflow)和重绘(Repaint)

发表于 2021-04-13 更新于 2022-12-11 分类于 浏览器

浏览器回流(Reflow)和重绘(Repaint)

回流必将引起重绘，重绘不一定会引起回流。

重绘

当页面中元素样式的改变并不影响它在文档流中的位置时（例如：color、background-color、visibility 等），浏览器会将新样式赋予给元素并重新绘制它，这个过程称为重绘。


回流

当 Render Tree (渲染树 由DOM树和CSS对象模型树组成)中部分或全部元素的尺寸、结构、或某些属性发生改变时，浏览器重新生成渲染树并重新进行布局渲染的过程称为回流

浏览器中的同源政策

同源指的是三个相同

  • 协议相同
  • 域名相同
  • 端口相同

当两个网页的URL中上述任意一个不同，则两个网页不同源

  • 不同源的网页被限制不能访问修改对方网页，即不同源的网页不能通信

跨域中的option请求

当使用CORS进行跨域时，CORS请求分为 简单请求和非简单请求

当发送跨域的非简单请求时，浏览器会在正式发送跨域请求前预先发送一个option方式的请求

option请求

options请求是浏览器自发起的**preflight request(预检请求)**，以检测实际请求的是否可以被服务器接受。

这个option方式的请求用来让服务端返回允许的请求方法（如get、post），允许跨域访问的客户端Origin（来源，或者域），还有是否需要Credentials(认证信息）

06-XSS攻击和CRSF攻击

发表于 2021-04-13 更新于 2022-12-11 分类于 浏览器

XSS攻击和CRSF攻击

XSS攻击

XSS, 即为（Cross Site Scripting）, 中文名为跨站脚本, 是发生在目标用户的浏览器层面上的，当渲染DOM树的过程中发生了不在预期内执行的JS代码时，就发生了XSS攻击。大多数XSS攻击的主要方式是嵌入一段远程或者第三方域上的JS代码。实际上是在目标网站的作用域下执行了这段js代码。

XSS攻击的危害

  1. 窃取cookie

    第三方的js代码运行在用户访问的域名下，可以通过 document.cookie 获得用户的 Cookie 信息。

  2. 未授权操作

    利用js的特性，代替用户在页面中进行一些操作，比如代替用户发送微博等

  3. 按键记录和钓鱼

    利用js记录用户输入的账号密码等，通过js修改DOM，伪造一个登录框或者链接诱导用户点击

浏览器中的进程与线程

以Chrome浏览器为例

进程与线程

1、进程

  • 进程是计算机中程序的一次运行活动，是系统进行资源分配的基本单位。通常我们启动一个程序，CPU便会开启一个进程，并给这个进程分配内存资源。
  • 任一时刻，一个CPU核心只能运行一个进程，其他进程处于挂起状态。（关于多进程，只是多个进程轮流占用CPU，且每次占用很短时间，这就是并行）
  • 进程之间相互独立，互不影响，一个进程不能直接访问其他进程中的资源。

2、线程

  • 线程是程序执行的最小单元。通常一个进程中包含了一个或多个线程。
  • 同一进程中的线程共享资源，且可以并行执行。
  • 某些线程之间存在相互制约的关系。

浏览器的垃圾回收机制

内存泄漏

程序的运行需要内存。只要程序提出要求，操作系统或者运行时（runtime）就必须供给内存。

对于持续运行的服务进程（daemon），必须及时释放不再用到的内存。否则，内存占用越来越高，轻则影响系统性能，重则导致进程崩溃。

不再用到的内存，没有及时释放，就叫做内存泄漏（memory leak）。
