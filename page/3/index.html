07-HTTP协议

发表于 2021-04-26 更新于 2021-05-24 分类于 前后端交互

HTTP协议

1.HTTP协议概念

HTTP(Hyper Text Transfer Protocol)：超文本传输协议,规定如何从服务器传输超文本文件(HTML文件)到浏览器，基于浏览器-服务器(B-S)架构工作，是客户端和服务器请求和应答的标准

超文本(HTML)：在文本中不仅仅能包含文字，还可以包含音频视频图像等资源

2.报文

HTTP请求和响应的数据块被称为报文，包括要传送的数据和一些附加信息，并且遵守规定好的格式

请求报文：浏览器发给服务器的HTTP请求报文

响应报文：服务器发给浏览器的HTTP响应报文

06-服务器端基本概念

发表于 2021-04-26 分类于 前后端交互

服务器端基本概念

1.URL

URL：统一资源定位符，标识internet网上资源的一种编码方式

使用URL可以唯一标识网络上的一个资源

URL组成：

  • 传输协议://服务器IP或域名:端口号/资源所在位置的标识
  • 例如: http://www.baidu.com/news/a.html
  • 即使用http协议访问域名为www.baidu.com，端口号为默认80的这个web服务器上的news文件夹下的a.html文件
  • http：超文本传输协议，一种发布和接收html文件的方法
  • IP或域名可以找到网络上的一台服务器，端口则代表这台服务器上的应用程序提供的服务，http默认端口为80
  • 资源所在位置标识：不一定是服务器上资源的路径，请求提交给服务器后，服务器可以根据位置标识做出其他的操作，服务器给客户端返回什么资源是服务器自身决定的

发表于 2021-04-26 更新于 2022-07-12 分类于 前端工程化

1. webpack概述

webpack是一个流行的前端项目构建工具（打包工具），提高开发效率和项目可维护性

好处：

  • 提供友好的模块化支持（ES6模块化）
  • 代码压缩混淆
  • 处理js兼容性问题
  • 性能优化
  • etc…

2. webpack基本使用

2.1 构建一个列表隔行变色的项目

  • 项目源代码都放在src文件夹中

项目目录

    • 运行index.js文件时因为使用了ES6模块化而报错
  • 使用webpack处理js文件后可以正常使用
    • 因为webpack提供了友好的ES6模块化支持

2.2 webpack安装

  • 运行npm run dev 命令后

    • webpack将项目中的index.js和之前通过npm命令安装的jquery.js一起打包为main.js文件
    • main.js文件位于项目根目录下的dist文件夹下 (/dist/main.js)

  • 此时在index.html文件中直接引用main.js文件，因为ES6模块化导致的错误消失（webpack处理js兼容性问题）

2.2.1 webpack配置项mode

mode可取值为

  • development ：开发模式，不压缩代码，不进行代码混淆，速度快

  • production：生产模式，压缩并混淆代码，速度慢


2.3 配置打包的入口与出口

webpack打包时有默认入口和出口

  • 默认入口：./src/index.js
  • 默认出口：./dist/main.js

可以在webpack.config.js文件中手动指定打包入口和出口

  • entry配置项指定打包的入口
  • output配置项指定打包后输出文件的名字和路径

2.4 配置自动打包功能

index.html文件中引用的是打包后输出的js文件

如果对js文件作出修改则需要重新打包生成输出文件，而每次手动打包很繁琐


使用自动打包功能可以监听文件的变化，当文件变化后自动打包

使用webpack-dev-server模块实现自动打包功能

2.4.1 使用webpack-dev-server插件

  1. 项目根目录下安装webpack-dev-server插件
  • npm install webpack-dev-server -D
  1. package.json文件下的scripts中新增命令如下

  1. 将index.html中js文件引用路径改为/bundle.js

  2. webpack.config.js中配置devServer配置项（将dist目录下的文件挂载到localhost:8080下）

    • static可以将路径设置为项目根目录（static:__dirname）

  3. 运行npm run start重新进行打包

  4. 浏览器访问http://localhost:8080查看index.html文件效果

    • 此时修改index.js文件并保存后，会自动打包并刷新index.html页面

注意：

  • webpack-dev-server启动一个实时打包的http服务器（访问服务器实时查看修改后的效果）

  • webpack-dev-server打包生成的输出文件（bundle.js）放在static配置项指定的目录下，而且存放于内存中，在硬盘中不可见

  • 当static配置项的值为dist文件夹下时，index.html文件要放在dist目录下，否则访问http://localhost:8080只能看到打包后的js文件


2.4.2 配置自动打包的相关参数

在package.json文件中scripts字段下的自动打包命令中添加参数即可

  • --open 运行自动打包命令后自动打开浏览器
  • --host 配置打开的http服务器的IP地址（默认为localhost/127.0.0.1）
  • --port 配置打开的http服务器的端口号（默认为8080）

配置服务器IP为127.0.0.1 运行于4000端口

2.6.3 打包处理css文件

  1. 安装style-loader和css-loader加载器
  • npm install style-loader css-loader -D
  1. 在webpack.config.js中新增module配置对象，在module对象下的rules数组中配置加载器匹配规则
  • ```js
    module:{

    rules:[

        //use调用对应加载器处理
        {test:/\.css$/,use:['style-loader','css-loader']}
    ]
}

2.6.7 打包处理样式表中图片和字体文件

但是在webpack5.58.0 版本中已经可以自动识别路径

对于旧版本可以使用url-loader和file-loader模块识别图片和字体文件的路径

2.6.7.1 安装使用url-loader和file-loader模块

    • base64格式的URL可以提升加载图片的速度

webpack默认不识别ES6的js高级语法，直接打包有ES6高级语法的js文件会报错

Vue脚手架

Vue脚手架：快速生成Vue项目基础架构，是一个命令行工具，官网地址：https://cli.vuejs.org/zh/

安装@vue/cli npm包来使用Vue脚手架

npm install -g @vue/cli

最新版的vue/cli同时支持使用基于2.x版本的脚手架创建旧vue项目

1.创建Vue项目

创建时报错

解决办法：

  • 管理员模式打开powershall，输入命令： set-ExecutionPolicy RemoteSigned，输入A后回车
  • 更改执行策略后解决问题

使用交互式命令行创建

  1. 在要创建的目录下输入命令 vue create '项目名称'

  • 创建的项目默认带Git仓库，通过vue create '项目名称' -n创建不带Git仓库的本地调试项目

  • 项目名称全英文、不能包含特殊字符('_','-'除外)


  1. 输入命令后显示选择界面，选择手动选择插件还是自动安装

  • 此处选择手动选择，使用键盘 上下键切换，回车选择


  1. 进行一系列选择

    • 选择安装的模块，此处选择babel，router(vue-router)，和Linter
      • babel：将新版本js代码转换为兼容性的旧版代码
      • router：vue路由模块
      • Linter：代码规则校验
    • 使用空格选中

    • 是否使用历史进行路由？此处选择否，我们使用hash进行前端路由

    • 选择ESLint语法版本，此处选择标准配置

    • 选择ESLint语法校验的时间，选择默认值

    • Babel，postCSS等模块的配置文件是单独创建还是整合到package.json中

    • 选择是否将此次手动配置保存为模板供下次创建使用

  1. 选择完毕

  • 项目创建完毕提示可以cd到目录下运行serve命令查看效果

  • 运行serve命令时自动使用webpack进行打包并打开http服务器供预览

  • 使用Local给出的URL进行预览


使用图形化界面创建

可以使用 vue ui在可视化图形界面中创建项目

运行vue ui

  • 包括模块管理等一些列操作都可以在图形化的页面中完成
