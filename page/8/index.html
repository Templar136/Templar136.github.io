07-Vue脚手架

发表于 2021-04-26 更新于 2022-07-12 分类于 前端工程化

Vue脚手架

Vue脚手架：快速生成Vue项目基础架构，是一个命令行工具，官网地址：https://cli.vuejs.org/zh/

安装@vue/cli npm包来使用Vue脚手架

npm install -g @vue/cli

最新版的vue/cli同时支持使用基于2.x版本的脚手架创建旧vue项目

1.创建Vue项目

创建时报错

解决办法：

  • 管理员模式打开powershall，输入命令： set-ExecutionPolicy RemoteSigned，输入A后回车
  • 更改执行策略后解决问题

使用交互式命令行创建

  1. 在要创建的目录下输入命令 vue create '项目名称'

  • 创建的项目默认带Git仓库，通过vue create '项目名称' -n创建不带Git仓库的本地调试项目

  • 项目名称全英文、不能包含特殊字符('_','-'除外)


  1. 输入命令后显示选择界面，选择手动选择插件还是自动安装

  • 此处选择手动选择，使用键盘 上下键切换，回车选择


  1. 进行一系列选择

    • 选择安装的模块，此处选择babel，router(vue-router)，和Linter
      • babel：将新版本js代码转换为兼容性的旧版代码
      • router：vue路由模块
      • Linter：代码规则校验
    • 使用空格选中

    • 是否使用历史进行路由？此处选择否，我们使用hash进行前端路由

    • 选择ESLint语法版本，此处选择标准配置

    • 选择ESLint语法校验的时间，选择默认值

    • Babel，postCSS等模块的配置文件是单独创建还是整合到package.json中

    • 选择是否将此次手动配置保存为模板供下次创建使用

  1. 选择完毕

  • 项目创建完毕提示可以cd到目录下运行serve命令查看效果

  • 运行serve命令时自动使用webpack进行打包并打开http服务器供预览

  • 使用Local给出的URL进行预览


使用图形化界面创建

上面使用命令行的方式创建项目时并不友好

可以使用 vue ui在可视化图形界面中创建项目

运行vue ui

会自动打开浏览器，在浏览器页面中进行项目的创建

  • 页面中还可以管理所有使用图形化界面创建的vue项目
  • 可以通过点击页面中的按钮，进行开发环境下项目的打包调试，或者进行生产环境下的打包
  • 包括模块管理等一些列操作都可以在图形化的页面中完成

创建基于Vue2.x版本的旧vue项目

  1. 安装@vue/cli-init模块
  • npm install @vue/cli-init -g
  1. 使用 vue init webpack '项目名称'命令创建项目
  • 在命令行中进行一系列配置后创建旧版项目

2.Vue脚手架生成项目的结构

  • 还有一些插件的配置文件如babel，postCSS的配置文件
  • src为源代码目录（程序员写的代码存放目录）

src目录下：

  • assets：资源（样式表，图片等）
  • components：组件
  • router：路由配置文件
  • views：视图文件
  • App.vue：项目根组件
  • main.js：项目使用webpack打包时的入口文件

3.自定义Vue项目配置

创建项目时，已经预先定义好了许多配置

可以在vue.config.js文件进行自定义配置

  • vue.config.js文件不存在时自行创建

  • 对devServer模块修改默认打开的端口号，设置打包完后自动打开浏览器访问服务器

  • 关闭eslint验证，eslint对于一些警告会直接报错。

遇见的问题

App.vue文件报错

上图红框报错称template字段下的内容必须放在一个根元素中

解决办法：

  1. 在vscode的设置中搜索eslint

  1. 将上图红框选项的对勾去掉

原理：vscode安装了Vetur 插件，它在检测 vue-html 模板时，使用了 eslint-plugin-vue 插件。


vue单文件组件中注释的问题

  • 对象键值前必须有空格
  • 注释// 后必须有一个空格或者tab缩进
  • ','后不能有多余空格
  • 注释最后不能有多余空格

Element-UI

Element-UI：一套为开发者、设计师和产品经理准备的基于 vue2.0 的桌面组件(component)库

Element-Plus：~基于vue3.0的组件库

使用它可以让程序员专注于项目的业务逻辑，省下了美化样式的工作量

官网:https://element.eleme.cn/#/zh-CN

使用

基于命令行手动安装Element-ui (vue2.0)

  • 在项目根目录下安装依赖包
  • 在项目的main.js文件中导入相关资源
    • 组件相关样式需要单独导入
  • 在Element-UI官网找到写好的组件代码复制
    • 此处复制一段按钮的组件代码
  • 添加到vue单文件组件App.vue中
  • 重新执行serve命令查看效果

基于命令行手动安装Element-Plus (vue3.0)

vue3.0下不能使用Element-ui，要使用Element-Plus

  1. 安装依赖包
  • npm install element-plus - S
  1. 在main.js文件中进行配置

基于图形化界面安装Element-UI

  • 查找插件

  • 配置插件按需导入
    • 红框为按需导入
    • 默认为全部导入(导致项目体积大)

  • 测试效果

JS执行机制

1.JS是单线程

  • JavaScript语言的一大特点就是单线程，同一时间只能做一件事。

这是因为JavaScript诞生的目的所致——JavaScript是为了处理页面中用户的交互，以及操作DOM而诞生的。

我们对一个DOM元素进行操作时，不能同时进行多种操作，而只能先添加，再修改，然后删除。

  • 单线程意味着任务需要排队，前一个任务结束，后一个任务才开始。这样如果JS的执行时间过长就会导致页面渲染不连贯。导致页面渲染呈现阻塞的感觉。
