02-v-if和v-show区别

发表于 2022-12-11 分类于 Vue

v-if和v-show的区别

  • v-if分支结构仅当满足条件时才会渲染

  • v-show无论取值都会渲染，当v-show取值为false时隐藏元素

  • 一般来说，v-if有更高的切换开销，而v-show有更高的初始渲染开销

如果需要非常频繁地切换，则使用v-show较好;

如果在运行时条件不太可能改变，则使用v-if 较好。

04-Vue的diff算法

发表于 2022-12-11 分类于 Vue

Vue的diff算法

由于在浏览器中直接操作DOM的代价 “较大”，所以Vue引入了虚拟DOM，虚拟DOM是对真实DOM的一种抽象描述。

虚拟DOM

  • 修改真实DOM可能会引起重绘和回流，过多的回流会影响性能

  • 更改虚拟DOM不会进行重绘和回流操作

  • 虚拟DOM就是把真实DOM转换为Javascript代码

  • 虚拟DOM进行多次更改后，一次性比较并修改真实DOM（修改发生了变化的局部DOM） ,在真实DOM中进行重绘和回流，以此来减少频繁修改真实DOM引起的大量重绘和回流，提高性能

03-组件data必须为函数

发表于 2022-12-11 分类于 Vue

组件的data必须为函数

注册组件时data为一个函数，函数的返回值为一个对象，对象中定义模板数据

一个组件可以有多个实例，如果data直接使用对象，则一个实例修改数据，其他实例也会一起被修改

  • data为函数时，每个实例可以维护一份被返回对象的独立的拷贝，不会互相影响

根组件的data可以为对象

因为根组件一般只有一个实例，无所谓多个实例互相影响

06-Vue的watch和computed的区别

发表于 2022-12-11 分类于 Vue

Vue的watch和computed的区别

计算属性computed

  1. 支持缓存，只有依赖数据发生改变，才会重新进行计算
  2. 不支持异步，当computed内有异步操作时无效，无法监听数据的变化
  3. computed 属性值会默认走缓存，计算属性是基于它们的响应式依赖进行缓存的，也就是基于data中声明过或者父组件传递的props中的数据通过计算得到的值
  4. 应用场景：如果一个属性是由其他属性计算而来的，这个属性依赖其他属性，是一个多对一或者一对一，一般用computed
  5. 如果computed属性属性值是函数，那么默认会走get方法；函数的返回值就是属性的属性值；在computed中的，属性都有一个get和一个set方法，当数据变化时，调用set方法。

01-var/let/const区别

发表于 2022-12-11 分类于 es6

var和let和const区别

var

var 声明变量 没有块级作用域 存在变量提升(将声明提升至函数或代码段顶部) 值可更改

let es6新增

let 声明变量 有块级作用域 不存在变量提升(只能先声明再使用) 值可更改

const es6新增

const 声明常量 有块级作用域 不存在变量提升(只能先声明再使用) 值不可更改

值不可更改(地址不可更改)：

  • 基本数据类型不可更改
  • 引用数据类型可以修改内部数据结构(但不能直接修改整体引用数据，会导致地址变化)
